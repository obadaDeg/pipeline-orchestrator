# Implementation Plan: Dashboard UI Test Suite

**Branch**: `007-dashboard-ui-tests` | **Date**: 2026-03-23 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/007-dashboard-ui-tests/spec.md`

## Summary

Add a three-tier test suite for the existing dashboard React application: isolated component tests (Vitest + React Testing Library), page-level integration tests with a mocked API layer (MSW v2), and a small set of end-to-end critical-path tests (Playwright). No changes to production source code are required. All tests are runnable via a single command and integrated into CI.

## Technical Context

**Language/Version**: TypeScript 5.4 (same as dashboard production code)
**Primary Dependencies**:
- *Test runner*: Vitest 2.x + `@vitest/coverage-v8` + `jsdom`
- *Component assertions*: `@testing-library/react` + `@testing-library/user-event` v14 + `@testing-library/jest-dom`
- *API mocking*: MSW v2 (`msw`) — `setupServer` from `msw/node`
- *E2E*: `@playwright/test` (Chromium only in CI)
- *Production stack*: React 18.3, React Router v6, TailwindCSS 3.4, Vite 5, Lucide React, highlight.js

**Storage**: N/A — no new storage; E2E tests use the existing PostgreSQL stack via Docker Compose
**Testing**: Vitest (component + integration), Playwright (E2E)
**Target Platform**: Node.js 20 LTS (test runner), Chromium (E2E browser)
**Project Type**: Test suite for an existing web application
**Performance Goals**: Component + integration suite completes in under 60 seconds (SC-001)
**Constraints**: Zero changes to production source code (SC-004 / FR-010)
**Scale/Scope**: 11 component test files + 5 page integration test files + 4 E2E spec files

E2E tests live at the repo root in `e2e/` to keep Playwright separate from the Vite build and avoid config conflicts. `vitest.config.ts` explicitly excludes `e2e/**` to prevent cross-contamination.
