# Implementation Plan: CI/CD Pipeline Improvements

**Branch**: `004-improve-cicd` | **Date**: 2026-03-22 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/004-improve-cicd/spec.md`

## Summary

Extend the GitHub Actions CI workflow (`.github/workflows/ci.yml`) with three improvements:
1. **Widen triggers** — fire on every branch push, not only `main`.
2. **Integration test job** — provision ephemeral Postgres 16 + Redis 7 service containers, then run `npm run test:integration` (which internally handles migrations via the test helper).
3. **Docker build job** — run `docker compose build` to verify the image builds on every push.

No new source files, schema changes, or API contracts are needed — this is a single-file YAML change with zero application code impact.

## Final CI Workflow Design

### Trigger change

```yaml
on:
  push:
    branches: ['**']       # was: [main]
  pull_request:
    branches: [main]
```

### New job: `test-integration`

```yaml
test-integration:
  name: Integration Tests
  runs-on: ubuntu-latest
  needs: [lint-and-typecheck]
  services:
    postgres:
      image: postgres:16
      env:
        POSTGRES_DB: webhook_test
        POSTGRES_USER: postgres
        POSTGRES_PASSWORD: postgres
      ports: ['5432:5432']
      options: >-
        --health-cmd pg_isready
        --health-interval 5s
        --health-timeout 5s
        --health-retries 5
    redis:
      image: redis:7
      ports: ['6379:6379']
      options: >-
        --health-cmd "redis-cli ping"
        --health-interval 5s
        --health-timeout 5s
        --health-retries 5
  env:
    DATABASE_URL: postgresql://postgres:postgres@localhost:5432/webhook_test
    REDIS_HOST: localhost
    REDIS_PORT: "6379"
    NODE_ENV: test
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-node@v4
      with:
        node-version: '20'
        cache: 'npm'
    - run: npm ci
    - run: npm run test:integration
```

**Note**: No separate migration step — `runTestMigrations()` in `beforeAll` handles this inside the test process.

### New job: `docker-build`

```yaml
docker-build:
  name: Docker Build
  runs-on: ubuntu-latest
  needs: [lint-and-typecheck]
  steps:
    - uses: actions/checkout@v4
    - run: docker compose build
```

### Final job dependency graph

```
lint-and-typecheck
├── build
├── test              (unit tests — unchanged)
├── test-integration  (NEW — real Postgres + Redis)
└── docker-build      (NEW — validates Dockerfile)
```
